# Beacon Implementation Guide

## Overview

This guide explains the implementation of the Beacon solution, which is a fully AOT-compatible rule execution engine. The Beacon solution is generated by the Pulsar compiler and can be deployed as a standalone application.

## Key Components

1. **CodeGenerator**: Updated generator that uses RuleGroupGeneratorFixed for proper AOT compatibility and SendMessage support.

2. **BeaconTemplateManager**: Generates the complete Beacon solution structure including:
   - Project files
   - Program entry point
   - Runtime services
   - Configuration files

3. **BeaconBuildOrchestrator**: Orchestrates the build process for the Beacon solution:
   - Generates code from parsed rules
   - Creates the solution structure
   - Builds the solution
   - Produces a standalone executable

4. **Redis Integration**: Complete implementation of Redis services for data access:
   - Connection management
   - Value retrieval and storage
   - Message publishing and subscribing
   - Health monitoring

5. **Temporal Rule Support**: Circular buffer implementation for temporal rules:
   - Generic object value support
   - Time-based filtering
   - Efficient memory usage

## Implementation Steps

### Step 1: Generate the Beacon Solution

Use the CodeGenerator to generate the Beacon solution:

```csharp
var codeGenerator = new CodeGenerator(logger);
var generatedFiles = codeGenerator.GenerateCSharp(rules, buildConfig);
```

### Step 2: Create the Solution Structure

Use the BeaconTemplateManager to create the solution structure:

```csharp
var templateManager = new BeaconTemplateManager(logger);
var solutionFiles = templateManager.CreateSolutionStructure(outputPath);
```

### Step 3: Build the Solution

Use the BeaconBuildOrchestrator to build the solution:

```csharp
var orchestrator = new BeaconBuildOrchestrator(logger);
orchestrator.Build(outputPath, runtimeId);
```

## How to Use

To generate a Beacon solution, run:

```bash
dotnet run --project Pulsar.Compiler -- generate --rules ./TestRules.yaml --config system_config.yaml --output ./BeaconOutput
```

To build the generated solution:

```bash
cd BeaconOutput/Beacon
dotnet build
```

To build a standalone executable:

```bash
cd BeaconOutput/Beacon
dotnet publish -c Release -r linux-x64 --self-contained true
```

## AOT Compatibility

The generated Beacon solution is fully AOT-compatible, with the following enhancements:

1. Proper AOT compatibility attributes in Program.cs
2. Trimming configuration in trimming.xml
3. No dynamic code generation or reflection
4. Runtime services implemented using interfaces and dependency injection
5. Circular buffer implementation for temporal rules
